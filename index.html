01 OVERVIEW

What Are Agentic Frameworks?

An agentic framework is a software toolkit that provides the scaffolding for building autonomous AI agents — systems that can perceive, reason, plan, act via tools, and learn from observations in a loop. Rather than building the orchestration, memory management, tool routing, and error handling from scratch, frameworks give you pre-built primitives that handle the hard parts.

The ecosystem has exploded since 2023. From a handful of early libraries, we now have dozens of frameworks spanning the spectrum from low-level graph engines to high-level role-playing crew abstractions. Each makes fundamentally different architectural choices that shape what's easy, what's possible, and what's painful.

03 ANATOMY

Common Architecture Patterns

Despite their diversity, every agentic framework must solve the same set of core problems. The differences lie in how they solve them — and those design choices ripple through everything from developer experience to production reliability.

🔄

The Agent Loop

Every framework implements some variant of Perceive → Reason → Act → Observe. The key differentiator is whether this loop is implicit (framework-managed) or explicit (developer-controlled via graphs or state machines).

📐

State Management

How intermediate results, plans, and conversation history are tracked across steps. Ranges from simple message lists to typed state objects with reducer functions for concurrent merges.

🔧

Tool Abstraction

The interface between the LLM and external functions. Most frameworks use JSON Schema definitions. Some use native function calling, others use text-based parsing. MCP is emerging as a universal tool protocol.

🤝

Agent Communication

How multiple agents coordinate. Models include: hierarchical (supervisor delegates), peer-to-peer (shared message bus), sequential pipeline (chain of responsibility), and conversation (agents chat with each other).

Three Schools of Thought

1

Graph-First (LangGraph)

Model workflows as directed graphs with typed state. Nodes are computation steps, edges carry conditional routing logic. Maximum control and debuggability, but steeper learning curve. You reason about your agent the way you'd reason about a state machine.

2

Role-First (CrewAI)

Define agents by their role, goal, and backstory — then let the framework handle orchestration. Intuitive for business workflows where you think in terms of "the researcher, the analyst, the writer." Lower barrier to entry, less fine-grained control.

3

Conversation-First (AutoGen/AG2)

Agents are participants in a conversation. Multi-agent coordination happens through structured dialogue, debate, and consensus-building. Natural for scenarios involving iterative reasoning, peer review, or group decision-making.

04 LANGGRAPH

LangGraph — Graph-Based State Machines

Open Source Python TypeScript

LangGraph is the low-level workhorse of the LangChain ecosystem. It models agent workflows as directed graphs with explicit nodes, edges, state transitions, and conditional routing. It reached v1.0 in late 2025 and has become the default runtime for all LangChain agents.

Core Concepts

Nodes

Individual computation steps — an LLM call, a tool invocation, a data transformation. Each node receives the current state and returns updates.

Edges

Connections between nodes. Conditional edges inspect state to decide the next node — this is where branching and loop-back logic lives.

State

A typed object (typically a TypedDict or Pydantic model) that flows through the graph. Reducer functions merge concurrent updates safely.

Key Strengths

Durable execution — agents can persist through failures and resume automatically. Human-in-the-loop support lets you inspect and modify state at breakpoints. Comprehensive memory spanning short-term working memory and long-term persistent stores. Full integration with the LangChain ecosystem of 700+ tool connectors.

// Simplified LangGraph workflow structure
from langgraph.graph import StateGraph, END

# Define typed state
class AgentState(TypedDict):
    messages: list[BaseMessage]
    next_step: str

# Build graph
graph = StateGraph(AgentState)
graph.add_node("reason", reason_node)
graph.add_node("act", tool_node)
graph.add_node("respond", response_node)

# Conditional routing
graph.add_conditional_edges("reason", route_fn,
    {"tool_call": "act", "done": "respond"})
graph.add_edge("act", "reason")   # loop back
graph.add_edge("respond", END)

app = graph.compile(checkpointer=MemorySaver())

Best For

Production-grade agents requiring fault tolerance, precise state management, and complex branching logic. Teams already invested in LangChain who need durable, long-running workflows with human oversight.

05 CREWAI

CrewAI — Role-Based Agent Teams

Open Source Python Enterprise Tier

CrewAI models multi-agent collaboration as a team ("crew") of role-playing agents. You define each agent's role, backstory, and goal, then assemble them into a crew with a set of tasks. The agents communicate naturally and can delegate work to each other.

Two Architecture Modes

Crews (Autonomous)

Agents have true agency — they decide when to delegate, when to ask questions, and how to approach their tasks. Best for exploratory, creative, or research-oriented workflows.

Flows (Event-Driven)

Production-oriented pipelines with more predictability. Tasks execute in defined sequences with explicit handoff points. Better for workflows that need repeatability and auditability.

// CrewAI role-based pattern
from crewai import Agent, Task, Crew

researcher = Agent(
    role="Senior Research Analyst",
    goal="Find comprehensive data on the topic",
    backstory="Expert analyst with 10 years experience...",
    tools=[search_tool, scrape_tool]
)

writer = Agent(
    role="Technical Writer",
    goal="Transform research into clear documentation",
    backstory="Award-winning technical communicator..."
)

crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, writing_task],
    verbose=True
)
result = crew.kickoff()

Key Strengths

Fastest time to "hello world" — the role/goal/backstory pattern is immediately intuitive. Built-in task replay lets you re-run specific tasks with tweaked prompts. Enterprise features include observability, a visual studio (CrewAI Studio), and a paid control plane for deployment and monitoring.

Best For

Teams wanting an intuitive way to model agent teams with clear roles. Business workflow automation, content pipelines, and research workflows where the "who does what" is the natural way to think about the problem.

06 AUTOGEN / AG2

AutoGen / AG2 — Conversational Multi-Agent

Open Source Python

Originally created by Microsoft Research, AutoGen pioneered multi-agent conversation patterns. In late 2024, the project transitioned to the independent AG2 community. The core idea: agents are participants in structured conversations, and complex tasks emerge from their dialogue.

Core Concepts

Conversable Agents

Every agent can send and receive messages. Types include AssistantAgent (LLM-backed), UserProxyAgent (human or code executor), and custom agents.

Group Chat

Multiple agents participate in structured discussions. A GroupChatManager controls turn-taking, termination conditions, and topic routing.

Code Execution

Built-in sandboxed code execution lets agents write, run, and debug code collaboratively — ideal for data analysis, prototyping, and research.

Key Strengths

Excellent for iterative reasoning where agents critique and improve each other's work. Supports diverse chat topologies — two-agent, group, nested conversations. AutoGen Studio provides a no-code visual interface. Fully open source with no paid tier.

Considerations

The transition from AutoGen to AG2 fragmented some community resources. Best suited for prototyping and research; production deployment requires more manual infrastructure work compared to LangGraph or CrewAI Enterprise.

07 OPENAI AGENTS SDK

OpenAI Agents SDK — Minimalist & Production-Ready

Open Source Python TypeScript Mar 2025

Released in March 2025 as the successor to the experimental Swarm SDK, the OpenAI Agents SDK takes an opinionated, minimalist approach. Clean abstractions, built-in tracing, guardrails, and a focus on making simple things simple.

Core Primitives

Agents

LLM-backed units with instructions, tools, and optional guardrails. Configured with a model, system prompt, and tool list.

Handoffs

First-class support for passing control between agents. One agent can hand off to another when the task shifts domain — a triage agent routing to specialists.

Guardrails

Built-in validation and safety mechanisms. Input guardrails check user messages; output guardrails validate agent responses before delivery.

// OpenAI Agents SDK — handoff pattern
from agents import Agent, Runner

billing_agent = Agent(
    name="Billing Specialist",
    instructions="Handle billing inquiries...",
    tools=[lookup_invoice, process_refund]
)

triage = Agent(
    name="Triage",
    instructions="Route to the right specialist.",
    handoffs=[billing_agent, tech_agent, sales_agent]
)

result = await Runner.run(triage, "I need a refund")

Key Strengths

Lowest boilerplate of any framework — a working agent in under 10 lines. Built-in tracing and debugging with visualisation tools. Despite the OpenAI branding, it supports 100+ LLM providers via compatible endpoints. First-class tools include web search, file search, and code interpreter.

Considerations

Tightest integration and best experience is with OpenAI models. For complex stateful workflows with durable execution, LangGraph remains a better fit. Focused on practical agent systems rather than deeply complex orchestration.

08 EMERGING FRAMEWORKS

Other Notable Frameworks

Pydantic AI Rising

Built by the Pydantic team. Type-safe, validation-first agent development. If you love Pydantic's approach to data modelling, this brings the same rigour to agent outputs. Extremely clean API, excellent for structured extraction workflows.

Google ADK Cloud

Agent Development Kit announced at Google Cloud NEXT 2025. Optimised for Gemini but model-agnostic via LiteLLM. Unique capabilities include bidirectional audio/video streaming. Seamless Vertex AI deployment.

Smolagents (HuggingFace)

Lightweight, research-oriented framework from HuggingFace. Emphasises "code agents" that write Python code as their action space rather than JSON tool calls. Minimal dependencies, easy to hack on.

Strands Agents (AWS)

Built around MCP for massive tool access. Designed for AWS and already used internally at Amazon. Supports streaming, parallel agents, and cloud-native deployment patterns.

OpenAgents

Purpose-built for agent interoperability via native MCP + A2A protocol support. Agents from different frameworks can discover and collaborate through a shared network. The "internet of agents" vision.

MetaGPT

Domain-specific multi-agent framework for software development. Agents play roles like Product Manager, Architect, and Engineer. Generates full project structures from natural language specs.

09 PROTOCOLS

MCP & A2A — The Interoperability Layer

Two emerging protocols are reshaping how agentic frameworks connect to the outside world and to each other.

🔌

Model Context Protocol (MCP)

Developed by Anthropic, MCP standardises how LLMs connect to external tools and data sources. Think of it as a "USB-C for AI" — one protocol that replaces bespoke integrations for every tool.

An MCP server exposes tools, resources, and prompts via a standard JSON-RPC interface. Any MCP-compatible agent can discover and call these tools without custom code. Major frameworks (LangGraph, CrewAI, Strands, Google ADK) now support MCP natively.

🤝

Agent-to-Agent Protocol (A2A)

Proposed by Google, A2A defines how agents communicate with each other — regardless of which framework built them. It covers capability discovery, task delegation, status updates, and result sharing.

While MCP connects agents to tools, A2A connects agents to other agents. Together, they form the foundation for a truly interoperable agent ecosystem where a LangGraph agent can collaborate with a CrewAI agent via standard protocols.

11 CHOOSING

How to Choose a Framework

Walk through the decision tree below. Your answers will guide you toward the right framework for your use case.

Summary Recommendations

→

Need precise state control & durability?

LangGraph. The graph-based model gives you fine-grained control over complex, stateful workflows with built-in persistence and human-in-the-loop.

→

Want intuitive role-based agent teams?

CrewAI. Fastest setup, role-based abstraction is easy to reason about. Great for business workflow automation and content pipelines.

→

Building conversational multi-agent debates?

AG2 (AutoGen). Diverse chat patterns, group decision-making, iterative reasoning. Fully free and open-source.

→

Want minimal boilerplate with great DX?

OpenAI Agents SDK for the OpenAI stack, or Pydantic AI for type-safe, model-agnostic simplicity.

→

Need cross-framework agent interoperability?

OpenAgents with native MCP + A2A. Purpose-built for persistent agent networks where agents from different frameworks discover and collaborate.
